Fitting and Plotting a Random Forest Model with PySpark on EC2

Set up PySpark

With the help of PySpark EC2 Setup, you should have set up a virtual environment that has PySpark installed.

Start the virtual environment:

source ~/pyspark-venv/bin/activate

Install any missing plotting libraries:

pip install pandas matplotlib

Spark’s DataFrame-based MLlib API expects features to be packed into a single vector column. In PySpark, VectorAssembler is the usual tool for combining multiple numeric feature columns into one vector column. Spark’s RandomForestRegressor then trains a random forest regression model using that feature vector.

Create a working folder

On the EC2 instance:

mkdir -p ~/pyspark-random-forest-demo
cd ~/pyspark-random-forest-demo

We will write one script:

rf_housing_demo.py

Run from Jupyter instead

Start Jupyter on EC2:

source ~/pyspark-venv/bin/activate

jupyter notebook \
  --no-browser \
  --ip=127.0.0.1 \
  --port=8888

From your local machine, create an SSH tunnel (again, refer to the Connect SSH info):

ssh -i my-key.pem -NL 8888:127.0.0.1:8888 ec2-user@EC2_PUBLIC_IP

Then open:

http://127.0.0.1:8888

From the EC2 terminal, copy the “token” in the URL. This is your password to get into the local terminal Jupyter notebook.

What we want to build

Spark session

spark = (
    SparkSession.builder
    .appName("ec2-random-forest-regression-demo")
    .master("local[*]")
    .config("spark.sql.shuffle.partitions", "8")
    .getOrCreate()
)

local[*] tells Spark to use all available CPU cores on the EC2 instance.

This is not a multi-node Spark cluster. It is single-machine Spark.

Synthetic data

The synthetic dataset has:

price =
    base price
    + bedroom effect
    + bathroom effect
    + square-footage effect
    - age penalty
    - distance penalty
    + noise

The random forest model should usually discover that sqft, distance_to_city_miles, and age_years matter.

VectorAssembler

Spark MLlib models usually do not accept many separate feature columns directly. Instead, they expect a single vector column.

This:

assembler = VectorAssembler(
    inputCols=feature_cols,
    outputCol="features",
    handleInvalid="skip",
)

turns columns like this:

bedrooms | bathrooms | sqft | age_years | distance_to_city_miles

into one column like this:

features

The resulting features column is what RandomForestRegressor requires.

Random forest model

The model is defined here:

rf = RandomForestRegressor(
    featuresCol="features",
    labelCol="price",
    predictionCol="prediction",
    numTrees=80,
    maxDepth=8,
    seed=42,
)

A random forest is an ensemble of decision trees. Each tree is created with a subset of the full dataset. The final prediction averages across the trees, which usually makes the model more stable than a single decision tree.

Important parameters:

Parameter Meaning
numTrees Number of trees in the forest
maxDepth Maximum depth of each tree
featuresCol Feature vector column
labelCol Target column
predictionCol Output prediction column
seed Makes results easier to reproduce

Pipeline

The pipeline joins preprocessing and modeling:

pipeline = Pipeline(stages=[assembler, rf])
model = pipeline.fit(train_df)

Due to the pipeline usage, the fitted model contains both:

  1. The trained feature assembler step.
  2. The trained random forest model.

Evaluation

The script computes:

RMSE
MAE
R^2

Plotting

Converting to Pandas

Spark DataFrames are distributed. Matplotlib expects local arrays.

So this is the common pattern:

plot_pdf = (
    predictions
    .select("price", "prediction")
    .sample(fraction=0.25, seed=7)
    .limit(1000)
    .toPandas()
)

Do not call .toPandas() on a huge Spark DataFrame.

Instead:

  1. Select only the columns you need.
  2. Sample or limit the rows.
  3. Convert the small result to pandas.
  4. Plot locally.

Actual vs. predicted plot

The diagonal line represents perfect predictions.

Points near the line are good.

Points far from the line are mistakes.

plt.scatter(plot_pdf["price"], plot_pdf["prediction"], alpha=0.35)
plt.plot([min_val, max_val], [min_val, max_val], linestyle="--")

If the model is learning well, the scatter should form a rough diagonal band.

Feature importance plot

Random forest models expose feature importances:

rf_model = model.stages[-1]
importances = rf_model.featureImportances.toArray()

Because the random forest is the last stage in the pipeline, model.stages[-1] retrieves the fitted random forest model.

Then we match each importance score back to the original feature name:

importance_pdf = pd.DataFrame(
    {
        "feature": feature_cols,
        "importance": importances,
    }
)

Copy plots back to your laptop

From your local machine, run:

scp -i my-key.pem \
  ec2-user@EC2_PUBLIC_IP:~/pyspark-random-forest-demo/outputs/*.png \
  .

This copies the generated PNG files into your current local directory.

Practical EC2 tips

  • Use enough memory
  • Avoid pulling huge results into pandas

This is safe:

predictions.limit(1000).toPandas()

This can crash your instance:

predictions.toPandas()
  • Keep outputs organized

A simple folder layout:

pyspark-random-forest-demo/
├── rf_housing_demo.py
├── data/
└── outputs/
    ├── actual_vs_predicted.png
    ├── feature_importances.png
    └── rf_housing_pipeline_model/
  • Stop the EC2 instance when finished

When you are done experimenting, stop or terminate the instance to avoid unnecessary cost.

(Appendix) Full Random Forest on EC2 PySpark script

Create the script:

cat > rf_housing_demo.py <<'PY'
from pathlib import Path

import matplotlib.pyplot as plt
import pandas as pd

from pyspark.sql import SparkSession
from pyspark.sql.functions import col, rand, round as spark_round

from pyspark.ml import Pipeline
from pyspark.ml.feature import VectorAssembler
from pyspark.ml.regression import RandomForestRegressor, RandomForestRegressionModel
from pyspark.ml.evaluation import RegressionEvaluator


# ---------------------------------------------------------------------
## Start Spark
# ---------------------------------------------------------------------

spark = (
    SparkSession.builder
    .appName("ec2-random-forest-regression-demo")
    .master("local[*]")
    .config("spark.sql.shuffle.partitions", "8")
    .getOrCreate()
)

spark.sparkContext.setLogLevel("WARN")

output_dir = Path("outputs")
output_dir.mkdir(exist_ok=True)


# ---------------------------------------------------------------------
## Build a synthetic housing dataset
# ---------------------------------------------------------------------
#
# Imagine each row is a home listing.
#
# Features:
# - bedrooms
# - bathrooms
# - sqft
# - age_years
# - distance_to_city_miles
#
# Label:
# - price
#
# The price formula below is intentionally simple, but includes noise.
# A random forest should learn the broad pattern without us telling it
# the formula.

n = 5000

df = (
    spark.range(0, n)
    .withColumn("bedrooms", (1 + (rand(seed=1) * 5)).cast("int"))
    .withColumn("bathrooms", spark_round(1 + rand(seed=2) * 3, 1))
    .withColumn("sqft", spark_round(650 + rand(seed=3) * 3350, 0))
    .withColumn("age_years", spark_round(rand(seed=4) * 80, 0))
    .withColumn("distance_to_city_miles", spark_round(rand(seed=5) * 40, 1))
    .withColumn("noise", spark_round((rand(seed=6) - 0.5) * 50000, 2))
    .withColumn(
        "price",
        spark_round(
            50000
            + col("bedrooms") * 18000
            + col("bathrooms") * 22000
            + col("sqft") * 210
            - col("age_years") * 1200
            - col("distance_to_city_miles") * 3500
            + col("noise"),
            2,
        ),
    )
    .drop("noise")
)

print("\nSample data:")
df.show(5, truncate=False)

print("\nSchema:")
df.printSchema()


# ---------------------------------------------------------------------
## Define features and label
# ---------------------------------------------------------------------

feature_cols = [
    "bedrooms",
    "bathrooms",
    "sqft",
    "age_years",
    "distance_to_city_miles",
]

label_col = "price"

assembler = VectorAssembler(
    inputCols=feature_cols,
    outputCol="features",
    handleInvalid="skip",
)


# ---------------------------------------------------------------------
## Split into train and test data
# ---------------------------------------------------------------------

train_df, test_df = df.randomSplit([0.8, 0.2], seed=42)

print(f"\nTraining rows: {train_df.count()}")
print(f"Test rows:     {test_df.count()}")


# ---------------------------------------------------------------------
## Fit a random forest regressor
# ---------------------------------------------------------------------
#
# A few important parameters:
#
# - numTrees: number of decision trees in the forest
# - maxDepth: maximum depth of each tree
# - seed: makes the demo repeatable
# - featuresCol: vector column created by VectorAssembler
# - labelCol: target column we want to predict

rf = RandomForestRegressor(
    featuresCol="features",
    labelCol=label_col,
    predictionCol="prediction",
    numTrees=80,
    maxDepth=8,
    seed=42,
)

pipeline = Pipeline(stages=[assembler, rf])

model = pipeline.fit(train_df)

print("\nModel fitted.")


# ---------------------------------------------------------------------
## Score the test set
# ---------------------------------------------------------------------

predictions = model.transform(test_df)

print("\nPredictions:")
predictions.select(
    "bedrooms",
    "bathrooms",
    "sqft",
    "age_years",
    "distance_to_city_miles",
    "price",
    "prediction",
).show(10, truncate=False)


# ---------------------------------------------------------------------
## Evaluate model quality
# ---------------------------------------------------------------------

rmse_evaluator = RegressionEvaluator(
    labelCol=label_col,
    predictionCol="prediction",
    metricName="rmse",
)

r2_evaluator = RegressionEvaluator(
    labelCol=label_col,
    predictionCol="prediction",
    metricName="r2",
)

mae_evaluator = RegressionEvaluator(
    labelCol=label_col,
    predictionCol="prediction",
    metricName="mae",
)

rmse = rmse_evaluator.evaluate(predictions)
r2 = r2_evaluator.evaluate(predictions)
mae = mae_evaluator.evaluate(predictions)

print("\nEvaluation metrics:")
print(f"RMSE: ${rmse:,.2f}")
print(f"MAE:  ${mae:,.2f}")
print(f"R^2:  {r2:.4f}")


# ---------------------------------------------------------------------
## Convert a small sample to pandas for plotting
# ---------------------------------------------------------------------
#
# Spark is good for distributed processing.
# Matplotlib is local.
#
# So the usual pattern is:
#
## Use Spark to train and score the large dataset.
## Pull down only the small result needed for plotting.

plot_pdf = (
    predictions
    .select("price", "prediction")
    .sample(fraction=0.25, seed=7)
    .limit(1000)
    .toPandas()
)


# ---------------------------------------------------------------------
## Plot actual vs. predicted prices
# ---------------------------------------------------------------------

plt.figure(figsize=(8, 6))
plt.scatter(plot_pdf["price"], plot_pdf["prediction"], alpha=0.35)

min_val = min(plot_pdf["price"].min(), plot_pdf["prediction"].min())
max_val = max(plot_pdf["price"].max(), plot_pdf["prediction"].max())

plt.plot([min_val, max_val], [min_val, max_val], linestyle="--")
plt.xlabel("Actual price")
plt.ylabel("Predicted price")
plt.title("Random Forest: Actual vs. Predicted Housing Prices")
plt.tight_layout()

actual_vs_predicted_path = output_dir / "actual_vs_predicted.png"
plt.savefig(actual_vs_predicted_path, dpi=150)
plt.close()

print(f"\nSaved plot: {actual_vs_predicted_path}")


# ---------------------------------------------------------------------
## Plot feature importances
# ---------------------------------------------------------------------

rf_model = model.stages[-1]

importances = rf_model.featureImportances.toArray()

importance_pdf = (
    pd.DataFrame(
        {
            "feature": feature_cols,
            "importance": importances,
        }
    )
    .sort_values("importance", ascending=True)
)

plt.figure(figsize=(8, 5))
plt.barh(importance_pdf["feature"], importance_pdf["importance"])
plt.xlabel("Importance")
plt.title("Random Forest Feature Importances")
plt.tight_layout()

feature_importance_path = output_dir / "feature_importances.png"
plt.savefig(feature_importance_path, dpi=150)
plt.close()

print(f"Saved plot: {feature_importance_path}")


# ---------------------------------------------------------------------
## Save the model
# ---------------------------------------------------------------------

model_path = output_dir / "rf_housing_pipeline_model"

# Overwrite if re-running the script.
model.write().overwrite().save(str(model_path))

print(f"Saved model: {model_path}")


# ---------------------------------------------------------------------
## Load the model back
# ---------------------------------------------------------------------

loaded_model = model.load(str(model_path))
loaded_predictions = loaded_model.transform(test_df.limit(5))

print("\nPredictions from loaded model:")
loaded_predictions.select("price", "prediction").show(truncate=False)


# ---------------------------------------------------------------------
## Stop Spark
# ---------------------------------------------------------------------

spark.stop()
PY

Run it:

python rf_housing_demo.py

You should end up with this folder:

outputs/
├── actual_vs_predicted.png
├── feature_importances.png
└── rf_housing_pipeline_model/
