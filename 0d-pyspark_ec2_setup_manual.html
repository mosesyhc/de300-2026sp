Setting Up PySpark on EC2

A practical setup manual with helper scripts

Overview

This guide walks through setting up PySpark on a single Amazon EC2 instance. It is intended for development, testing, lightweight analysis, and small jobs.

For production-scale Spark workloads, consider Amazon EMR, AWS Glue, Databricks, or another managed Spark platform instead of hand-rolling a standalone EC2 setup.

1. Launch the EC2 instance

Use “Launch from template”.

Setting Recommendation
AMI Amazon Linux 2023
Instance type t2.large or larger
Storage 80 GB gp3
Security group SSH only from your IP
Key pair Create or select an existing .pem key

SSH into the instance.

Choose your instance \(\rightarrow\) Connect (SSH Client) \(\rightarrow\) Copy the ssh -i ... command:

chmod 400 my-key.pem

ssh -i my-key.pem ec2-user@EC2_PUBLIC_IP

2. Install Java, Python, and basic tools

Spark needs Java. On Amazon Linux 2023, Amazon Corretto 17 is a good default.

sudo dnf update -y

sudo dnf install -y \
  java-17-amazon-corretto-devel \
  python3 \
  python3-pip \
  git \
  wget \
  tar

Check that the basics are installed:

java -version
python3 --version
pip3 --version

3. Create a Python environment

Use a virtual environment so the PySpark setup does not get mixed into the system Python installation.

python3 -m venv ~/pyspark-venv
source ~/pyspark-venv/bin/activate

python -m pip install --upgrade pip
pip install pyspark jupyter pandas pyarrow

4. Test PySpark

Create a small test and save into test_pyspark.py:

cat > test_pyspark.py <<'PY'
from pyspark.sql import SparkSession

spark = (
    SparkSession.builder
    .appName("ec2-pyspark-test")
    .master("local[*]")
    .getOrCreate()
)

data = [
    ("Alice", 10),
    ("Bob", 20),
    ("Charlie", 30),
]

df = spark.createDataFrame(data, ["name", "score"])

df.show()
print("Row count:", df.count())

spark.stop()
PY

Run it:

python test_pyspark.py

Expected output should include a small table and:

Row count: 3

5. Helper script: full setup

Save this as setup_pyspark_ec2.sh.

#!/usr/bin/env bash
set -euo pipefail

echo "Updating packages..."
sudo dnf update -y

echo "Installing Java, Python, and utilities..."
sudo dnf install -y \
  java-17-amazon-corretto-devel \
  python3 \
  python3-pip \
  git \
  wget \
  tar

echo "Creating Python virtual environment..."
python3 -m venv "$HOME/pyspark-venv"

echo "Installing Python packages..."
source "$HOME/pyspark-venv/bin/activate"
python -m pip install --upgrade pip
pip install pyspark jupyter pandas pyarrow

echo "Writing PySpark smoke test..."
cat > "$HOME/test_pyspark.py" <<'PY'
from pyspark.sql import SparkSession

spark = (
    SparkSession.builder
    .appName("ec2-pyspark-test")
    .master("local[*]")
    .getOrCreate()
)

df = spark.createDataFrame(
    [("Alice", 10), ("Bob", 20), ("Charlie", 30)],
    ["name", "score"]
)

df.show()
print("Row count:", df.count())

spark.stop()
PY

echo "Running smoke test..."
python "$HOME/test_pyspark.py"

echo ""
echo "Done."
echo "To use PySpark later, run:"
echo "source ~/pyspark-venv/bin/activate"

Run it:

chmod +x setup_pyspark_ec2.sh
./setup_pyspark_ec2.sh

6. Helper script: run a PySpark job

For small projects, it is useful to keep a simple wrapper script around your jobs.

Save this as run_job.sh.

#!/usr/bin/env bash
set -euo pipefail

if [ "$#" -lt 1 ]; then
  echo "Usage: $0 path/to/job.py [args...]"
  exit 1
fi

source "$HOME/pyspark-venv/bin/activate"

JOB_FILE="$1"
shift

python "$JOB_FILE" "$@"

Example:

chmod +x run_job.sh
./run_job.sh jobs/example_job.py

7. Example PySpark job

Save this as jobs/example_job.py.

from pyspark.sql import SparkSession
from pyspark.sql import functions as F

spark = (
    SparkSession.builder
    .appName("example-job")
    .master("local[*]")
    .getOrCreate()
)

df = spark.createDataFrame(
    [
        ("A", 10),
        ("A", 15),
        ("B", 20),
        ("B", 25),
    ],
    ["group", "value"],
)

result = (
    df.groupBy("group")
    .agg(
        F.count("*").alias("rows"),
        F.avg("value").alias("avg_value"),
    )
    .orderBy("group")
)

result.show()

spark.stop()

8. Common troubleshooting

Java not found

Check Java:

java -version

If missing:

sudo dnf install -y java-17-amazon-corretto-devel

PySpark cannot find Python

Add this to ~/.bashrc:

export PYSPARK_PYTHON=python3

Reload the shell config:

source ~/.bashrc

EC2 runs out of memory

Use a larger instance or limit Spark driver memory:

spark = (
    SparkSession.builder
    .appName("memory-limited-job")
    .master("local[*]")
    .config("spark.driver.memory", "4g")
    .getOrCreate()
)

Disk fills up

Check disk usage:

df -h
du -sh ~/.cache /tmp 2>/dev/null

Clean package cache:

sudo dnf clean all

9. Final sanity checklist

Before calling the setup finished, run:

java -version
python3 --version
source ~/pyspark-venv/bin/activate
python -c "import pyspark; print(pyspark.__version__)"
python ~/test_pyspark.py

If all of those pass, PySpark is working on the EC2 instance.
